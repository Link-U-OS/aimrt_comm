## 前言

本仓库展示了 **AimRTe** 提供的各个功能的使用示例。

读者在上手阅读、修改、编译执行示例源码时，建议同时阅读本文档对应部分的描述。虽然我们尽量把各个程序的启动逻辑做到一致，但偶尔还是会有需要特殊说明的地方。

### 命名空间

本仓库的根命名空间为 **example::**，我们按功能划分成了多个章节，对应着 [example](.) 目录下的多个子目录，每个子目录的命名空间再以它们各自的目录名字，又深入了一层，如 example::module::

### 关于配置与脚本

若看到示例子目录下存在 install 相关的目录，它大概是启动该示例所需的配置文件或脚本。用户可以选择编译全部目标，使它们被部署到正确的地方（一般是 <u>你的 build 目录/bin</u> 中），也可以指定编译其中的某一个，如，以 [example/module/install](./module/install) 目录定义的部署目标为例，可以在你的 build 目录中执行以下语句：

```sh
cmake ..
make example-module-install-SETUP_ALL
```

来实现只部署该目标下的配置与脚本。

示例中的配置与脚本的路径多以相对目录来表示，其中，我们假定了相关示例目标编译后，会在 build 目录下呈现以下结构：

- **bin**：存放了编译输出的可执行程序，配置文件目录一般为 **cfg**（部分示例使用 **config**）；
- **lib**：存放了编译输出的库目标，包括 ros, protobuf, 其他第三方库生成的库目标。

配置中相关参数，如 ip 地址等，均可以根据实际情况调整（见各个示例的特殊说明）。
